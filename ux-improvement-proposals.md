# Alpha Trip UX 개선 제안서

## 1. 긴급 버그 수정 (Critical)

### 1-1. AI 출장 도우미 — 밋업 선택 드롭다운 무한 반복 버그

현재 AI 출장 도우미 페이지(`/chatbot`)에서 밋업 선택 영역이 동일한 3개 밋업(수정 테스트 밋업, 커스텀 수화물 밋업, 수화물 테스트 밋업)을 **수십 번 반복 렌더링**하여 화면 전체를 덮고 있습니다. 채팅 인터페이스가 거의 보이지 않을 정도로 심각한 상태이며, 사용자가 AI 도우미 기능을 사용할 수 없습니다. 이 버그는 최우선으로 수정해야 합니다.

---

## 2. 네비게이션 및 정보 구조 개선

### 2-1. 상단 네비게이션 간소화

현재 상단 네비게이션에 **8개 메뉴 항목 + 백오피스 + 유저 메뉴 + 언어 선택**이 나열되어 있어 시각적 과부하가 발생합니다. 99%의 사용자가 모바일로 접속하는 환경을 고려하면, 핵심 메뉴 4~5개만 노출하고 나머지는 "더보기" 또는 햄버거 메뉴로 묶는 것이 효과적입니다.

| 현재 구조 | 제안 구조 |
|---|---|
| 마이페이지, 밋업 신청, 여정표 조회, 항공편/픽업, 예약센터, AI 도우미, 커뮤니티, 백오피스 (8개) | **주요**: 밋업 신청, 여정표, 예약센터, AI 도우미 (4개) + **더보기 메뉴**: 마이페이지, 항공편/픽업, 커뮤니티, 백오피스 |

### 2-2. 모바일 하단 네비게이션 바 강화

모바일 환경에서 하단 네비게이션 바(MobileBottomNav)가 이미 존재하지만, 가장 자주 사용하는 기능 중심으로 재배치하면 접근성이 크게 향상됩니다. 홈, 밋업 신청, AI 도우미, 마이페이지 4개 탭으로 구성하고, 각 탭에 뱃지(알림 수, 미완료 항목 수)를 표시하면 사용자 참여를 유도할 수 있습니다.

## 3. 홈페이지 개선

### 3-1. 비로그인 사용자를 위한 랜딩 페이지

현재 홈페이지는 로그인 여부와 관계없이 동일한 화면을 보여줍니다. 비로그인 사용자에게는 서비스 소개, 핵심 가치 제안, 회원가입 유도 CTA를 강조하는 별도의 랜딩 페이지를 제공하면 전환율이 크게 향상됩니다.

### 3-2. 기능 카드 우선순위 구분

현재 12개 기능 카드가 모두 동일한 크기와 스타일로 나열되어 있어 사용자가 어디서부터 시작해야 할지 혼란스럽습니다. 핵심 기능(밋업 신청, 예약센터, AI 도우미)은 **대형 카드**로, 보조 기능(텔레그램 알림, 데이터 분석 등)은 **소형 카드**로 시각적 위계를 만들면 사용자 동선이 명확해집니다.

### 3-3. 히어로 섹션 비주얼 강화

"환영합니다, OOO님" 텍스트와 3개 버튼만 있는 현재 히어로 섹션에 **다음 밋업 일정 카운트다운**, **최근 활동 요약**, **프로필 완성도 프로그레스 바** 등 개인화된 대시보드 요소를 추가하면 재방문 동기가 강화됩니다.

## 4. 회원가입 및 온보딩 플로우 개선

### 4-1. 프로그레시브 온보딩

현재 회원가입 후 Welcome 페이지에서 다음 단계를 안내하지만, 실제 프로필 완성까지의 여정이 끊깁니다. **스텝 바이 스텝 온보딩 위저드**(이름 → 프로필 사진 → 여권 정보 → 선호 설정)를 도입하면 프로필 완성률이 크게 높아집니다.

### 4-2. 마이페이지 프로필 완성도 게이지

마이페이지 상단에 "온보딩 미완료" 뱃지만 표시되어 있는데, **원형 프로그레스 바**(예: "프로필 35% 완성")와 함께 "다음으로 전화번호를 입력하세요" 같은 구체적 행동 유도 메시지를 추가하면 사용자가 자연스럽게 정보를 채우게 됩니다.

---

## 5. 커뮤니티 페이지 개선

### 5-1. 빈 상태(Empty State) 디자인

현재 커뮤니티 페이지에 대화방이 없으면 "전체 채팅방" 텍스트만 보이고 나머지는 완전히 빈 화면입니다. 일러스트레이션과 함께 "아직 대화방이 없습니다. 첫 번째 대화방을 만들어 보세요!" 같은 안내 메시지와 CTA 버튼을 추가해야 합니다.

### 5-2. 기본 채팅방 자동 생성

서비스 가입 시 "공지사항", "자유게시판", "여행 팁 공유" 같은 기본 채팅방을 자동으로 생성하면 커뮤니티가 활성화되어 보이는 효과가 있습니다.

---

## 6. 밋업 신청 폼 개선

### 6-1. 폼 진행률 표시

밋업 신청 폼이 상당히 길어서(교통수단 → 필수 정보 → 식사 → 추가 정보) 사용자가 중간에 이탈할 수 있습니다. 상단에 **스텝 인디케이터**(1/4 → 2/4 → 3/4 → 4/4)를 추가하고, 각 섹션을 별도 스텝으로 분리하면 완료율이 높아집니다.

### 6-2. 자동 채우기(Auto-fill)

로그인한 사용자의 이름, 전화번호, 메신저 ID 등은 마이페이지 프로필에서 자동으로 채워주면 반복 입력을 줄일 수 있습니다.

---

## 7. 전반적인 UX 품질 개선

### 7-1. 로딩 상태 및 스켈레톤 UI

페이지 전환 시 빈 화면이 잠깐 보이는 경우가 있습니다. 모든 데이터 로딩 구간에 **스켈레톤 UI**를 적용하면 체감 속도가 빨라집니다.

### 7-2. 토스트 알림 일관성

성공/에러 메시지의 표시 위치와 스타일을 통일하고, 중요한 액션(밋업 신청 완료, 프로필 저장 등)에는 **성공 애니메이션**을 추가하면 사용자 만족도가 높아집니다.

### 7-3. 다크/라이트 모드 전환 개선

현재 다크 모드 전환 버튼이 있지만, 일부 페이지(로그인 페이지 등)에서 테마 불일치가 발생할 수 있습니다. 전체 페이지에서 일관된 테마 적용을 검증해야 합니다.

### 7-4. PWA 설치 팝업 타이밍 조정

현재 모든 페이지에서 PWA 설치 팝업이 즉시 표시됩니다. 사용자가 서비스를 2~3회 이상 방문하거나 특정 기능을 사용한 후에 표시하면 설치 전환율이 더 높아집니다.
