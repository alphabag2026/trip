# Alpha Trip - 플랫폼 기능 소개서

**문서 버전:** v9.4 | **작성일:** 2026년 3월 29일 | **도메인:** alphatrip.org

---

## 1. 플랫폼 개요

Alpha Trip은 글로벌 밋업 주최자(B)가 참석자(C/G)를 초청하고, 항공권/호텔 예약부터 현장 운영까지 전 과정을 자동화하는 **올인원 출장 관리 플랫폼**입니다. 특히 USDT 기반 크립토 결제를 핵심으로 하여, 각국 부가세(VAT)를 절감한 가격으로 항공권과 호텔을 예약할 수 있는 차별화된 결제 시스템을 제공합니다.

| 항목 | 내용 |
|------|------|
| **기술 스택** | React 19 + Tailwind 4 + Express 4 + tRPC 11 + Drizzle ORM |
| **데이터베이스** | MySQL (TiDB) - 66개 테이블 |
| **서버 API** | 60+ tRPC 라우터, 6,000+ 줄 서버 코드 |
| **프론트엔드** | 30+ 페이지, 35+ 관리자 페이지 |
| **인증** | 이메일/비밀번호 + Manus OAuth + 2FA(TOTP) |
| **배포** | Vultr VPS (Docker) - alphatrip.org |
| **다국어** | 한국어 기본, 영어/중국어/일본어 등 지원 |

## 2. 사용자 유형 및 역할 체계

Alpha Trip은 4단계 역할 기반 접근 제어(RBAC)를 구현하고 있으며, 각 역할에 따라 접근 가능한 기능이 달라집니다.

| 역할 | 코드 | 설명 | 주요 권한 |
|------|------|------|-----------|
| **슈퍼 관리자** | `superadmin` | 플랫폼 전체 운영자 | 모든 기능, 사용자 관리, 감사 로그, 광고 배너 |
| **주최자** | `organizer` | 밋업 주최 업체/개인 | 밋업 생성, 참가자 관리, 파트너 연결, 예산 관리 |
| **에이전시** | `agency` | 여행사/대행사 | 파트너 관리, 예약 대행, 수익 트래킹 |
| **파트너** | `partner` | 식당/클럽/차량/통역 등 | 서비스 제공, 예약 확인, 매출 조회 |
| **일반 사용자** | `user` | 밋업 참석자 (C/G) | 밋업 등록, 여행 정보 조회, 예약, 채팅 |

## 3. 인증 및 회원 관리

### 3.1 회원가입 시스템

플랫폼은 개인 사용자와 비즈니스 사용자(주최자/에이전시/파트너)를 구분하는 **이중 가입 체계**를 운영합니다. 비즈니스 계정은 가입 시 회사 정보(사업자등록번호, 업종, 직원 수, 설립 연도 등)를 입력하며, 관리자 승인 워크플로우를 거쳐 활성화됩니다.

| 기능 | 상세 |
|------|------|
| **이메일 가입** | 이메일/비밀번호 기반 가입, 이메일 인증(Resend API) |
| **Manus OAuth** | 소셜 로그인 연동 |
| **2FA (TOTP)** | Google Authenticator 등 TOTP 기반 2단계 인증 |
| **비밀번호 재설정** | 이메일 기반 토큰 발급 → 재설정 |
| **주최자 승인** | 비즈니스 계정 가입 → 관리자 검토 → 승인/거절 |
| **온보딩 가이드** | 단계별 온보딩 (이메일 인증 → 프로필 → 여권 → 첫 밋업) |

### 3.2 사용자 프로필 및 여권 관리

사용자는 상세 프로필 정보(국적, 생년월일, 소속, 직책, 비상연락처, 식이제한, 알레르기 등)를 등록할 수 있으며, **여권 정보는 1회 등록 후 모든 밋업에서 자동 연동**됩니다. 여권 OCR 인식 기능을 통해 여권 사진을 업로드하면 자동으로 정보가 추출됩니다.

| DB 테이블 | 용도 |
|-----------|------|
| `users` | 기본 계정 정보 (이메일, 역할, 2FA 설정) |
| `user_profiles` | 상세 프로필 (국적, 직책, 건강정보, 텔레그램 ID) |
| `passport_info` | 여권 정보 (여권번호, 만료일, OCR 데이터) |
| `onboarding_progress` | 온보딩 진행률 추적 |

## 4. 밋업 관리 시스템

### 4.1 밋업 생성 및 운영

주최자는 밋업을 생성하고, 참가자를 초청하며, 일정/숙소/교통편/식사를 통합 관리할 수 있습니다.

| 기능 | 설명 |
|------|------|
| **밋업 생성** | 제목, 설명, 장소, 기간, 최대 인원, 예산 설정 |
| **참가자 등록** | 온라인 등록 폼 + QR 코드 체크인 |
| **일정 관리** | 시간대별 이벤트 스케줄 (장소, 설명, 담당자) |
| **숙소 배정** | 호텔 객실 배정 및 바우처 발급 |
| **교통편 관리** | 항공편/버스/기차 등 교통편 옵션 및 참가자 배정 |
| **비용 관리** | 카테고리별 지출 내역 (항공, 호텔, 식사, 선물 등) |
| **수정 요청** | 참가자의 정보 변경 요청 → 관리자 승인 |

### 4.2 초청 시스템

밋업 초청은 이메일, SMS, 링크, CSV 일괄 업로드 4가지 방식을 지원하며, 초청 템플릿을 미리 설정하여 반복 사용할 수 있습니다.

| 기능 | 설명 |
|------|------|
| **이메일 초청** | Resend API를 통한 이메일 발송 |
| **링크 초청** | 고유 토큰 기반 초청 링크 생성 |
| **CSV 일괄 초청** | CSV 파일 업로드로 대량 초청 |
| **초청 통계** | 발송/열람/수락/거절 비율 실시간 추적 |
| **초청 템플릿** | 조직별 이메일/SMS 템플릿 관리 |

## 5. 항공권 예약 시스템 (Mystifly GDS 연동)

### 5.1 Mystifly API 연동

Alpha Trip은 **Mystifly GDS API**를 통해 전 세계 항공권을 실시간으로 검색하고 예약할 수 있습니다. Mystifly는 Amadeus, Sabre, Galileo 등 주요 GDS를 통합 제공하는 B2B 항공권 플랫폼입니다. 현재 데모 모드로 운영 중이며, Mystifly API 계정 발급 시 실제 GDS 데이터로 자동 전환됩니다.

| API 엔드포인트 | 기능 |
|----------------|------|
| `CreateSession` | Mystifly 인증 세션 생성 |
| `AirSearch` | 항공편 검색 (편도/왕복, 좌석 등급, 인원) |
| `FareRevalidation` | 선택한 항공편 가격 재검증 |
| `FareRules` | 운임 규정 조회 (환불/변경 정책) |
| `BookFlight` | 항공편 예약 (승객 정보 + PNR 생성) |
| `IssueTicket` | 발권 (관리자 전용) |
| `CancelBooking` | 예약 취소 (관리자 전용) |

### 5.2 가격 비교 시스템

항공권 및 호텔 검색 결과에서 **현지통화 / USD / USDT** 3가지 가격을 동시에 표시하며, USDT 결제 시 부가세(VAT) 절감 금액을 하이라이트합니다.

| 가격 항목 | 계산 방식 |
|-----------|-----------|
| **현지 가격** | 해당 국가 통화 기준 원가 (VAT 포함) |
| **USD 가격** | 현지 가격 / 환율 |
| **USDT 가격** | (현지 가격 - VAT) x 환율 + 환전수수료(1.85%) |
| **절감액** | 현지 가격 - USDT 가격 (VAT 절감분) |

20개국 VAT 데이터가 사전 등록되어 있으며, 국가별 부가세율(7%~25%)에 따라 절감액이 자동 계산됩니다.

## 6. USDT 결제 시스템

### 6.1 결제 수단

Alpha Trip은 3가지 결제 수단을 지원하며, 사용자가 예약 시 선택할 수 있습니다.

| 결제 수단 | 설명 | 수수료 |
|-----------|------|--------|
| **Direct USDT** | 지정 지갑 주소로 직접 USDT 전송 후 TX 해시 입력 | 0% |
| **NOWPayments** | NOWPayments 게이트웨이를 통한 자동 결제 확인 | 0.5% |
| **Platform Balance** | 플랫폼 내 충전 잔액으로 즉시 결제 | 0% |

### 6.2 블록체인 네트워크 지원

| 네트워크 | 토큰 | 특징 |
|----------|------|------|
| **TRC20** (Tron) | USDT | 가장 저렴한 수수료 (~$1) |
| **BEP20** (BSC) | USDT | 빠른 확인 속도 |
| **ERC20** (Ethereum) | USDT | 가장 높은 유동성 |
| **Polygon** | USDT | 저렴한 수수료 + 빠른 속도 |

### 6.3 결제 플로우

```
사용자 항공편/호텔 선택
    |
결제 수단 선택 (Direct USDT / NOWPayments / Platform Balance)
    |
네트워크 선택 (TRC20 / BEP20 / ERC20 / Polygon)
    |
[Direct USDT] 지갑 주소 표시 → 사용자 전송 → TX 해시 입력 → 관리자 확인
[NOWPayments] Invoice 생성 → 결제 페이지 리다이렉트 → Webhook 자동 확인
[Platform Balance] 잔액 차감 → 즉시 확인
    |
예약 확정 → 발권 (관리자)
```

### 6.4 플랫폼 지갑 시스템

각 사용자에게 플랫폼 내 USDT 지갑이 제공되며, 충전/결제/환불/보너스 등 모든 거래 내역이 기록됩니다.

| DB 테이블 | 용도 |
|-----------|------|
| `payment_transactions` | 결제 트랜잭션 (NOWPayments/직접전송/자체화폐) |
| `platform_wallets` | 사용자별 지갑 잔액 (잔액, 동결금액, 총입금, 총지출) |
| `wallet_transactions` | 지갑 입출금 내역 (입금/출금/결제/환불/보너스/이체) |
| `payment_gateway_config` | 결제 게이트웨이 설정 (수수료, 지갑 주소, 한도) |

## 7. 호텔 바우처 및 항공권 티켓 관리

### 7.1 호텔 바우처

관리자가 참가자별 호텔 예약확인서(바우처)를 등록하면, 참가자는 모바일에서 바우처를 확인하고 호텔 체크인 시 제시할 수 있습니다.

| 정보 항목 | 내용 |
|-----------|------|
| 호텔명 (영문/현지어) | 현지어 호텔명 및 주소 동시 표시 |
| 예약 정보 | 체크인/아웃 날짜, 객실 유형, 인원 |
| 부가 정보 | 식사 포함 여부, 특별 요청, 취소 정책 |
| 위치 정보 | 위도/경도 (지도 연동) |
| 파일 첨부 | 바우처 이미지/PDF 업로드 |

### 7.2 항공권 티켓

왕복 항공권 정보를 구조화하여 관리하며, 출발편/귀국편 각각의 항공사, 편명, 공항, 시간, 좌석 등급, 수하물 정보를 포함합니다.

## 8. 커뮤니티 채팅 시스템

### 8.1 실시간 채팅

밋업별 또는 주제별 채팅방을 생성하여 참가자 간 실시간 소통이 가능합니다.

| 기능 | 설명 |
|------|------|
| **채팅방 유형** | 일반, 공지, 지원, 소셜, 1:1, 그룹 |
| **메시지 유형** | 텍스트, 이미지, 파일, 시스템, 공지, 영상, 위치, 음성 |
| **위치 공유** | 위도/경도 + 장소명 공유 |
| **자동 번역** | 채팅방별 자동 번역 활성화 (다국어 참가자 지원) |
| **메시지 고정** | 중요 메시지 핀 고정 |
| **답글** | 특정 메시지에 대한 답글 기능 |
| **WebRTC** | 음성/영상 통화 지원 |

### 8.2 AI 챗봇

LLM 기반 AI 챗봇이 밋업 관련 질문에 자동 응답합니다. 일정, 숙소, 교통편, 현지 정보 등을 자연어로 질문하면 즉시 답변을 제공합니다.

## 9. 텔레그램 연동

### 9.1 텔레그램 봇 기능

| 기능 | 설명 |
|------|------|
| **알림 발송** | 밋업 일정 변경, 체크인 알림, 긴급 공지 |
| **브로드캐스트** | 전체/그룹별 일괄 메시지 발송 |
| **여행정보 자동 업로드** | 텔레그램으로 받은 항공편/호텔 정보를 LLM이 자동 파싱 |
| **파싱 결과 검토** | 관리자가 LLM 파싱 결과를 검토 → 승인 → 시스템 반영 |

텔레그램 업로드 기능은 텔레그램에서 전송된 텍스트/이미지/문서를 LLM이 자동으로 분석하여 항공편, 호텔, 일정, 교통편 등으로 분류하고 구조화된 데이터로 변환합니다. 파싱 신뢰도(0~100%)가 함께 표시되어 관리자가 검토 후 승인할 수 있습니다.

## 15. 외부 REST API

외부 시스템과의 연동을 위한 REST API를 제공합니다. API 키 기반 인증, 시간당 요청 제한(Rate Limit), 요청 로그 기록 기능을 포함합니다.

| 항목 | 내용 |
|------|------|
| **인증** | API 키 (SHA-256 해시 저장) |
| **Rate Limit** | 기본 1,000 req/hour (키별 설정 가능) |
| **권한** | 키별 접근 권한 설정 (registrations:read, meetups:read 등) |
| **로그** | 모든 API 요청 기록 (메서드, 엔드포인트, 상태코드, 응답시간) |
